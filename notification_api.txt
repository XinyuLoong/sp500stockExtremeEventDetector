Assumptions:

1. Assume that the S&P 500 constituents is consistent while we monitoring it.
   According to S&P Dow Jones Indices, constituent changes are made on an 
   as-needed basis in response to market developments such as mergers or 
   delistings, while index maintenance activities such as quarterly share 
   count updates follow a regular schedule. But here I take the Wikipedia
   table as the editable source of truth.
2. Assume that Wikipedia table is updated. 
   For the one-time stock-list harvesting script, I use the publicly available 
   Wikipedia S&P 500 constituents table. After stocklist.csv is generated, 
   stocklist.csv is treated as the editable source of truth for the monitoring 
   program.
3. Current_price is defined as the latest available intraday bar close from 
   yfinance, not the official daily closing price. Because the program is 
   scheduled hourly during trading time, this value is used as a proxy for the 
   latest available market price.
4. Assume that the "N days" in the requirement refers to calendar day rather 
   than trading day.
5. Trading time is defined as the NYSE regular trading session, 9:30 AM to 
   4:00 PM Eastern Time, excluding non-trading days and market holidays. Use 
   pandas_market_calendars to check whether the market is open.


Notification API Description:

The notification.py has two functions: format_notification_line(event: dict) and 
send_notifications(events: list[dict], window_days: int, threshold: float, 
output_file: str = "notifications",).

(1) format_notification_line(event: dict) -> str
This function write one line for a stock price alert event to a sentence given one 
event.
The input var is a dict (event), whose structure is like this:
{
    "symbol": symbol of one stock
    "company_name": company name of this stock
    "current_price": current price at polling time of this stock
    "reference_price": the max or min price in the past N days
    "event_type": RISE or DROP, indicating the price is extremely high or low
    "change_pct": the change percent of price
    "message": a short message which describes this event
}
And the output is a formatted line based on the event information above:
[company name]([symbol])'s price rose by [pct(%)] compared to the recent 
[maximum/minimum] price.
e.g. "Apple Inc. (AAPL)'s price rose by 15.00% compared to the recent maximum price."

(2) send_notifications(events: list[dict], 
                       window_days: int, 
                       threshold: float, 
                       output_file: str = "notifications") -> void
This function doesn't have a return. It will write a notification log under the 
current working directory.
It will write the header based on the inputs, and has a for loop to call 
format_notification_line for each event.
Then it concatenates the header and lines generated by format_notification_line.

The final notification format is:
    Timestamp: [timestamp]
    Check Window: [window_days] day(s)
    Threshold: [threshold]%
    [space line]
    [space line]
    [company name] ([symbol])'s price [rose/dropped] by [pct]% compared to the 
    recent [maximum/minimum] price.
    ...
The input var is a list of event described above. Eg. [event1, event2, event3, ...]
It also needs the inputs like:
    -window_days: the arg we passed from the command line, means how many days we want to check
    -threshold: the arg we passed from in the command line, it decides when we will see a rise/drop as extreme event
    -output_file: a base string to name the output notification log fiile. It's default value is "notifications"
    Based on the output_file, we create the real file name, formatted as:
    [output_file]_YYYYMMDD_HHMM.log



How it works:

The notification module is called only after main.py finishes price polling and 
event detection.
The workflow is:
1. main.py loads the stock list from stocklist.csv.
2. main.py polls recent intraday price data for each stock.
3. main.py calculates the current price, recent maximum price, and recent minimum price for each stock.
4. main.py detects events based on the configured threshold：
    A RISE event is detected when: (current_price - window_max) / window_max > threshold
    A DROP event is detected when: (current_price - window_min) / window_min < -threshold
5. If at least one event is detected, main.py calls the notification API by:
    from notification import send_notifications
    send_notifications(events, args.days, args.threshold)
6. send_notifications() receives the list of detected events and creates a notification log file.
7. For each event, send_notifications() calls format_notification_line() to convert the event dictionary into a human-readable alert sentence.
8. The notification module writes the timestamp, check window, threshold, and all formatted alert messages into a local .log file.
9. If no event is detected, main.py does not call the notification API and no notification log file is created.
