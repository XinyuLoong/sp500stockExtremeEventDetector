# S&P 500 Stock Extreme Event Detector

## Overview

This project monitors S&P 500 stock prices and generates notifications when a stock price moves beyond a configurable threshold from its recent extreme price.

The program is designed to be executed once per hour by a cron job. However, price polling only happens during regular NYSE trading hours. If the market is closed, the program exits without calling the price data provider.

The application supports:

- Harvesting the S&P 500 stock list into an editable CSV file
- Reading the monitored stock list from stocklist.csv
- Polling recent intraday stock price data using yfinance
- Detecting extreme price movement events based on configurable parameters
- Writing current price and extreme information to a state CSV file
- Sending notifications through a separate notification module
- Running automatically through a crontab file

## Core Interpretation of the Requirement

This implementation interprets the event detection rule as an extreme breakout rule:

- A RISE event is triggered when the current price is more than the configured threshold above the recent maximum price.
- A DROP event is triggered when the current price is more than the configured threshold below the recent minimum price.

Formula:

```text
RISE event:
(current_price - window_max) / window_max > threshold

DROP event:
(current_price - window_min) / window_min < -threshold
```

## Design Logic and Implementation Notes

### 1. Stock List Source

For production use, the official S&P Dow Jones Indices source would be preferred.

For this coding test, the one-time stock-list harvesting script uses the publicly available Wikipedia S&P 500 constituents table. After stocklist.csv is generated, the CSV file is treated as the editable source of truth for the monitoring program.

The monitoring program does not automatically refresh the S&P 500 constituent list during each hourly run. If the stock list changes, stocklist.csv can be manually edited or regenerated by running get_stock.py again.

### 2. Trading Time Check

The cron job runs every hour, every day. However, the program should only poll prices during trading time. main.py checks market hours internally.

The implementation uses pandas_market_calendars to check whether the NYSE market is open. This allows the program to consider:

- Regular NYSE trading hours
- Weekends
- Market holidays
- Non-trading days

The assumed regular trading session is:

```text
9:30 AM to 4:00 PM Eastern Time
```

If the market is closed, the program exits before polling prices.

For testing outside trading hours, the program supports:

```bash
--force-run
```

### 3. Current Price Definition

Since the program is scheduled to run hourly during trading time, the current price is defined as the latest available intraday bar close returned by yfinance.

It is not the official daily closing price.

In this implementation:

```text
current_price = latest available 5-minute intraday Close price
```

This value is used as a proxy for the latest available market price at polling time.

### 4. Recent Window Definition

The requirement mentions the “last N days.” This implementation interprets N days as recent calendar days.

The program calls yfinance.download() with:

```python
period = f"{days}d"
interval = "5m"
```

Then it calculates the recent maximum and minimum from the available regular-session intraday bars returned within that period.

The main values are:

```text
current_price = latest available intraday Close price
window_max = maximum intraday High price in the returned window
window_min = minimum intraday Low price in the returned window
```

### 5. Price Data Provider

The current implementation uses yfinance to fetch intraday stock price data.

During testing, fetching all S&P 500 tickers in one request sometimes returned incomplete or inconsistent data for some symbols. The missing data issue appeared to be inconsistent across runs.

To reduce this issue, the program fetches stock prices in smaller batches:

```text
50 tickers per batch
```

This batching approach produced more consistent results during short-term testing, so the implementation assumes that batched requests significantly reduce the chance of missing price data.

### 6. Ticker Symbol Conversion

Some ticker symbols are represented differently across data sources.

For example:

```text
Wikipedia: BRK.B
Yahoo Finance: BRK-B
```

To handle this, the program converts symbols containing "." into Yahoo Finance-compatible symbols by replacing "." with "-" before polling prices.

After polling, the program maps Yahoo symbols back to the original symbols from stocklist.csv.

### 7. Notification Module Design

The notification logic is implemented in a separate file:

```text
notification.py
```

This keeps event detection and notification delivery separated.

The main program only needs to call:

```python
from notification import send_notifications

send_notifications(events, args.days, args.threshold)
```

The current implementation writes notifications to a local .log file.

## How to Run the Main Program

Basic command:

```bash
python main.py
```

By default, the program uses:

```text
--days 10
--threshold 0.10
--stock-file stocklist.csv
--state-file stock_extremes.csv
```

Equivalent explicit command:

```bash
python main.py --days 10 --threshold 0.10 --stock-file stocklist.csv --state-file stock_extremes.csv
```



## Command-Line Arguments

| Argument | Description | Default |
|---|---|---|
| --days | Number of recent calendar days used to calculate recent high and low prices | 10 |
| --threshold | Price movement threshold. For example, 0.10 means 10% | 0.10 |
| --stock-file | Input CSV file containing the stock list | stocklist.csv |
| --state-file | Output CSV file storing current stock prices and extremes | stock_extremes.csv |
| --force-run | Forces the program to run outside trading hours. Used for testing | False |

## Testing Outside Trading Hours

The program normally exits when the market is closed.

To test the program outside trading hours, use:

```bash
python main.py --force-run
```

## Output Files

### 1. State CSV File

The main program writes the current monitoring state to:

```text
stock_extremes.csv
```

This file stores information such as:

- Stock symbol
- Company name
- Current price
- Recent maximum price
- Recent minimum price
- Drop percentage
- Rise percentage
- Latest detected event
- Check timestamp

The state file is updated each time the program successfully runs and polls price data.

### 2. Notification Log File

If one or more events are detected, the notification module creates a local log file.

The file name format is:

```text
notifications_YYYYMMDD_HHMMSS.log
```


The notification log contains:

```text
Timestamp: [timestamp]
Check Window: [window_days] day(s)
Threshold: [threshold]%


[company name] ([symbol])'s price [rose/dropped] by [pct]% compared to the recent [maximum/minimum] price.
...
```

If no event is detected, no notification log file is created.

## Notification API

The notification module is implemented in:

```text
notification.py
```

It exposes two functions:

```python
format_notification_line(event: dict) -> str
```

and

```python
send_notifications(
    events: list[dict],
    window_days: int,
    threshold: float,
    output_file: str = "notifications"
) -> None
```

### format_notification_line(event: dict) -> str

This function converts one detected stock price event into a human-readable notification sentence.

Expected event dictionary structure:

```python
{
    "symbol": "AAPL",
    "company_name": "Apple Inc.",
    "current_price": 123.45,
    "reference_price": 110.00,
    "event_type": "RISE",
    "change_pct": 0.1227,
    "message": "A short internal message describing the event"
}
```

Example output:

```text
Apple Inc. (AAPL)'s price rose by 12.27% compared to the recent maximum price.
```

### send_notifications(...) -> None

This function writes all detected events into a local notification log file.

It receives:

- A list of event dictionaries
- The check window in days
- The threshold used for event detection
- An optional output file base name

The function does not return a value.

## Cron Job

The project includes a crontab file:

```text
stock_monitor.crontab
```

The cron job runs once every hour:

```crontab
0 * * * * cd /Users/longxinyu/sp500stockExtremeEventDetector && /Users/longxinyu/miniconda3/envs/iai/bin/python main.py --days 10 --threshold 0.10 --stock-file stocklist.csv --state-file stock_extremes.csv >> cron_run.log 2>&1
```

The program itself checks whether the market is open, so the cron job can run every hour every day.

A reusable path template is also included in the crontab file:

```bash

```

## Main Workflow

The full workflow is:

1. Run get_stock.py once to generate stocklist.csv.
2. Edit stocklist.csv if needed.
3. Set the desired --days and --threshold parameters.
4. Run main.py manually or through cron.
5. main.py checks whether the NYSE market is open.
6. If the market is open, the program polls recent intraday price data from yfinance.
7. The program calculates current_price, window_max, and window_min.
8. The program detects RISE and DROP events based on the configured threshold.
9. If events are detected, notification.py writes them into a notification log file.
10. The program writes the latest state into stock_extremes.csv.
